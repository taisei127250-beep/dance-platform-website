# Luma vs Street Dance Navi - 利用規約・プライバシーポリシー比較分析

## 📋 エグゼクティブサマリー

LumaとStreet Dance Naviの法務文書を比較分析し、追加すべき項目・不要な項目を理由とともに洗い出しました。

---

## 🔍 利用規約（Terms of Service）の比較

### ✅ **Street Dance Naviに追加すべき項目**

#### 1. **コンテンツライセンスの明確化** 【重要度: 高】

**Lumaの記載:**
> ユーザーが投稿するコンテンツについて、「無制限、世界規模、取消不可、非独占的、ロイヤリティ無料の権利」を取得

**Street Dance Naviの現状:**
> 「投稿コンテンツを無償で非独占的に使用する権利（サービス運営、プロモーション等の目的）を許諾」

**推奨:**
```
【追加すべき】
Lumaのような包括的なライセンス条項を明記すべき。特に：
- 「取消不可」の明記（ユーザーが削除しても運営が使用した実績は残る）
- 「世界規模」の明記（国際展開の可能性）
- 「サブライセンス権」の明記（SNS連携等で必要）
```

**理由:**
- ユーザーが投稿したイベント画像をSNSで共有する際に必要
- プラットフォームの成長に伴い、提携先への再配信が発生する可能性
- 法的リスクの低減

---

#### 2. **プラットフォームの役割の明確化** 【重要度: 高】

**Lumaの記載:**
> Lumaは「第三者サービス」として機能し、イベント主催者によるキャンセルや、イベント関連のコンテンツ・活動について責任を負わない

**Street Dance Naviの現状:**
> 第6条で「イベントの開催可否、内容変更等は、イベント主催者の責任」と記載

**推奨:**
```
【追加すべき】
利用規約の冒頭（第1条～第2条あたり）に、プラットフォームの役割を明記：

「当サービスの役割」
1. 当サービスは、イベント主催者と参加者をつなぐプラットフォームを提供します
2. イベントの企画・実施・管理は主催者の責任であり、当サービスは主催者ではありません
3. 当サービスは、イベントの品質・安全性・適法性について保証しません
4. 主催者と参加者間の紛争について、当サービスは仲裁者ではなく、責任を負いません
```

**理由:**
- イベントでトラブルが発生した際の責任範囲を明確化
- プラットフォーム vs 主催者の責任分界を明示
- 訴訟リスクの低減

---

#### 3. **輸出管理（Export Controls）** 【重要度: 低】

**Lumaの記載:**
> 米国輸出管理法に関する条項

**Street Dance Naviの現状:**
> 記載なし

**推奨:**
```
【追加不要】
日本国内サービスとして開始する場合、現時点では不要。
ただし、将来的に以下の場合は追加検討：
- 海外展開を行う場合
- 海外ユーザーの利用を想定する場合
```

---

#### 4. **紛争解決の時効期間** 【重要度: 中】

**Lumaの記載:**
> 請求は発生から1年以内に提出する必要がある

**Street Dance Naviの現状:**
> 記載なし

**推奨:**
```
【追加すべき】
第12条（準拠法・管轄裁判所）に追加：

「本規約に関する請求は、請求原因が発生した日から1年以内に
行う必要があります。この期間を過ぎた請求は受け付けません。」
```

**理由:**
- 長期間経過後の訴訟リスクを回避
- 証拠保全の負担軽減
- 一般的なSaaS規約のベストプラクティス

---

#### 5. **サービス変更・終了の詳細** 【重要度: 中】

**Lumaの記載:**
> サービス終了時の通知方法と期間を明記

**Street Dance Naviの現状:**
> 「サービスを終了する場合は、合理的な事前通知を行います」

**推奨:**
```
【具体化すべき】
第10条を以下のように変更：

「サービスの終了」
1. 当サービスは、サービスを終了する場合、少なくとも30日前に
   ウェブサイトおよびアプリ内で告知します
2. サービス終了時、ユーザーは自身のデータをエクスポートできます
3. エクスポート期間は終了日から30日間とします
```

**理由:**
- ユーザーの権利保護
- データポータビリティの確保
- 突然のサービス終了による混乱防止

---

### ❌ **Street Dance Naviに不要な項目**

#### 1. **複雑な損害賠償制限条項**

**Lumaの記載:**
> 詳細な損害賠償の制限（間接損害、利益損失等の除外）

**Street Dance Naviの現状:**
> シンプルな免責条項

**推奨:**
```
【現状維持でOK】
理由：
- 日本の消費者契約法では、事業者の免責条項が制限される
- 過度に複雑な条項は日本のユーザーに不親切
- 第9条の免責事項で十分カバーできている
```

## 🔒 プライバシーポリシーの比較

### ✅ **Street Dance Naviに追加すべき項目**

#### 1. **データ保持期間の具体化** 【重要度: 高】

**Lumaの記載:**
> 「データタイプに応じて異なる期間保持」と明記（具体的期間は未記載）

**Street Dance Naviの現状:**
> 「利用目的の達成に必要な期間のみ保管」

**推奨:**
```
【具体化すべき】
セクション「6. 情報の保管と管理」に追加：

「データ保持期間」
- アカウント情報: アカウント削除後30日間
- イベント参加履歴: イベント終了後3年間（税務対応のため）
- 決済記録: 取引完了後7年間（法令遵守のため）
- アクセスログ: 90日間
- その他のデータ: 最終利用から1年間

ユーザーからの削除請求があった場合、法令で保管が義務付けられている
データを除き、30日以内に削除します。
```

**理由:**
- GDPR対応（将来的な欧州展開の可能性）
- ユーザーの信頼獲得
- 法令遵守（電子帳簿保存法等）

---

#### 2. **第三者共有の詳細化** 【重要度: 高】

**Lumaの記載:**
> イベント主催者と参加者間での情報共有を明記
> 決済処理のためのStripe共有を明記

**Street Dance Naviの現状:**
> 外部サービス（Supabase、Stripe、Mapbox）の記載のみ

**推奨:**
```
【追加すべき】
セクション「4. 情報の第三者提供」を以下のように拡充：

「情報の共有」
1. イベント主催者への共有:
   - 参加者の氏名、メールアドレス、参加履歴を主催者と共有します
   - 主催者は参加者情報を、イベント運営目的のみに使用できます

2. 参加者への共有:
   - イベント主催者の名前、連絡先を参加者に開示します

3. 決済事業者との共有:
   - Stripeに対し、決済処理に必要な最小限の情報を提供します

4. その他のサービスプロバイダー:
   - データベース管理（Supabase）
   - 地図サービス（Mapbox）
```

**理由:**
- ユーザーの知る権利の尊重
- プラットフォームビジネスモデルの透明性
- トラブル発生時の説明責任

---

#### 3. **Cookie/トラッキングの詳細化** 【重要度: 中】

**Lumaの記載:**
> Google Analyticsの利用を明記
> オプトアウト方法を記載

**Street Dance Naviの現状:**
> Cookieの使用を簡潔に記載

**推奨:**
```
【追加すべき】
セクション「8. Cookie（クッキー）の使用」を拡充：

「使用するCookieの種類」
1. 必須Cookie: ログイン状態の維持（無効化不可）
2. 分析Cookie: Google Analytics（無効化可能）
   - オプトアウト: https://tools.google.com/dlpage/gaoptout

「トラッキング技術」
当サービスは、以下の技術を使用してサービスを改善します：
- セッション管理
- ユーザー行動分析
- エラー追跡（Sentry等）
```

**理由:**
- ePrivacy指令対応（EU）
- ユーザーのプライバシー意識向上への対応
- 透明性の確保

---

#### 4. **国際的なデータ転送** 【重要度: 中】

**Lumaの記載:**
> Google API関連の記載のみ

**Street Dance Naviの現状:**
> 記載なし

**推奨:**
```
【追加すべき】
新セクション「10. 国際的なデータ転送」を追加：

「データの保存場所」
当サービスは、以下のサービスプロバイダーを利用しており、
データが日本国外のサーバーに保存される場合があります：

- Supabase（米国）
- Stripe（米国）
- Mapbox（米国）

これらのサービスは、適切なセキュリティ対策を講じており、
GDPR等の国際基準に準拠しています。
```

**理由:**
- グローバルSaaSの利用実態を反映
- 個人情報保護法の改正対応
- ユーザーの信頼獲得

---

#### 5. **ユーザーの権利の拡充** 【重要度: 高】

**Lumaの記載:**
> アクセス権、修正権、削除権を明記
> 外部アカウント連携の取り消し権を明記

**Street Dance Naviの現状:**
> 開示、訂正、削除、利用停止の4権利のみ

**推奨:**
```
【追加すべき】
セクション「7. ユーザーの権利」に追加：

5. データポータビリティ権
   - 自身のデータを機械可読形式でエクスポートする権利
   - アプリ内「設定」→「データのエクスポート」から実行可能

6. 外部連携の取り消し権
   - ソーシャルログイン（Google、Apple等）の連携を
     いつでも取り消すことができます
   - 取り消し後も、メールアドレスでログイン可能です

7. 異議申し立て権
   - データ処理に異議がある場合、停止を求めることができます
```

**理由:**
- GDPR準拠
- ユーザーエンパワーメント
- 競合サービスとの差別化

## 💡 その他の推奨事項

### 1. **バージョン管理**

Lumaは規約に更新日を記載していますが、Street Dance Naviも同様です。
さらに強化するなら：

```
「更新履歴」
- 2025年1月15日: 第7条（決済）にプラットフォーム手数料を追記
- 2024年12月31日: 初版公開
```

### 2. **多言語対応**

将来的に訪日外国人ダンサーの利用を想定する場合：
- 英語版の規約・ポリシーを作成
- 「本規約は日本語を正文とし、翻訳版は参考訳です」と明記

### 3. **FAQ形式の補足**

規約やポリシーとは別に、わかりやすいFAQページを作成：
- 「私の個人情報は誰に共有されますか？」
- 「アカウントを削除したらデータはどうなりますか？」
- 「主催者は参加者の連絡先を見られますか？」

## 📝 まとめ

### Lumaから学ぶべき点：
1. **透明性**: データの流れ、共有先、保持期間を明確に
2. **ユーザー権利の尊重**: データポータビリティ、取り消し権の明記
3. **責任範囲の明確化**: プラットフォーム vs 主催者の責任分界

### Street Dance Naviが既に優れている点：
1. **未成年者保護**: 13歳未満の制限を明記
2. **日本法準拠**: 消費者保護法制に適合
3. **シンプルさ**: 過度に複雑でなく、読みやすい

### 最も重要な改善点：
**「プラットフォームの役割と責任範囲の明確化」**
→ これにより、イベントトラブル発生時の訴訟リスクを大幅に低減できます。
